# Data Sources & Attribution\n\nThis document records all external data and APIs used in the project for transparency, licensing, and reproducibility.\n\n## 1. Core Interaction Dataset\n- **Name:** (Kaggle) MyAnimeList Ratings Dataset (2019/2020 snapshot)\n- **URL:** https://www.kaggle.com/datasets/hernan4444/anime-recommendation-database-2020\n- **Contents:** User–anime explicit ratings (user_id, anime_id, rating, timestamp if available).\n- **License / Terms:** Refer to Kaggle dataset page; ensure compliance with uploader licensing.\n- **Usage:** Primary collaborative filtering training matrix. Provides historical interaction scale.\n- **Limitations:** Snapshot is static and does not include post-2020 titles or recent popularity shifts.\n\n## 2. Metadata / Enrichment (Jikan API)\n- **API:** Jikan (Unofficial MyAnimeList API)\n- **Base URL:** https://api.jikan.moe/v4/\n- **Endpoints Used:** `/anime/{id}/full` (plus any needed for seasonal or popular listings).\n- **Data Fields:** Titles, synopsis, genres, themes, demographics, studios, producers, source, episodes, status, airing dates, MAL score, rank, popularity, members.\n- **Usage:** Content-based features (genres/themes embeddings), synopsis text embeddings, popularity signal, cold-start handling for new titles absent in ratings dataset.\n- **Caching Strategy:** Raw JSON snapshots stored under `data/raw/jikan/` with retrieval timestamp. Monthly refresh suggested; old snapshots retained for reproducibility.\n- **Rate Limiting & Etiquette:** Throttle requests (~2–3 req/sec, adjust if 429 responses). Implement exponential backoff and reuse cached data where available.\n- **Limitations:** Unofficial service; occasional downtime or structural changes. Popularity/score values drift over time—version with timestamps.\n\n## 3. Derived Artifacts\n- **Processed Interaction Splits:** `data/processed/interactions_train.parquet`, `interactions_val.parquet`, `interactions_test.parquet` (user-aware or time-based split).\n- **Metadata Normalization:** `data/processed/anime_metadata.parquet` (canonical schema).\n- **Embeddings:** `data/processed/synopsis_embeddings.parquet` (or separate binary matrix).\n- **Feature Matrices:** Genre/theme multi-hot, TF-IDF vectors saved under `data/processed/features/`.\n\n## 4.
